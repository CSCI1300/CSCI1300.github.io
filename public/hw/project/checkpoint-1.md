# Final Project: Checkpoint 1

**Due:** Wednesday, July 1, 2026

Checkpoint 1 is about getting your **GitHub repository** ready. You are not expected to have code yet.

## Instructions

1. Create a **public** GitHub repository for your final project.
2. Add a **`README.md`** file to the repository.

Your `README.md` may be mostly empty for this checkpoint. You may use the template below with `(TBA)` in each section. By the **final project deadline**, it must describe your game (theme, goal, how to compile/run, how to play, classes, and any extra credit).

3. Add a **`ProjectDescription.md`**

Your `ProjectDescription.md` must include your theme, planned classes, planned limited resource, plan for handling time, any EC you hope to accomplish (and descriptions of how you plan to implement the EC option), your tradeoff system (such as JojaMart and the benefits of using JojaMart instead of other options), what mapping style you plan to use (just a description of what you want your map to look like is fine) and the win/lose condition.

Use headers for each requirement, in markdown files (.md) you can create headers using a \#, the more \#s you use, the smaller the header is. Use a single \# header for the header of the file, followed by two \#s for headers of each section. This is not autograded, so you don't have to use any specific words, but make sure your file is easy to read and understand for the course staff. You can use the README template below as a starting point for this file as well.

4. Push your repository to GitHub.
