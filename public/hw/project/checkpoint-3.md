Checkpoint 3 is a **major progress check** toward a complete project. You do not need every feature finished by this date, but you should be far enough along that you can finish before **your interview slot**.

## Requirements

Push to the **same public GitHub repository** you used for checkpoints 1 and 2.

Your project should show **substantial progress** on the required game features from sections 1–11. Aim for a playable draft: a running game loop, core classes implemented beyond stubs, and several major features working. It is OK if some pieces are still unfinished at checkpoint 3.

For checkpoint 3, your project must include:

1. A program that **compiles and runs** with `g++`.
2. A working **game loop** and **main menu** the player can interact with.
3. At least **3 classes**, each split into a `.h` and `.cpp` file.
4. At least **one class that inherits from another class**.
5. Core class methods implemented beyond empty stubs.
6. Meaningful progress toward the required final project features.
7. At least **one part of your game must read information from a file** using file input.
   * For example, your game might read a map, item list, character list, dialogue, starting resources, prices, events, or configuration values from a `.txt` file.
   * The file should be included in your GitHub repository.
   * Your code should actually use the file data in the running program, even if the feature is still simple.

Your project does not need to be perfect yet, but by checkpoint 3 it should look like a real draft of your final project rather than only a skeleton.
