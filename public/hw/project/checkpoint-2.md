**Due:** Wednesday, July 8, 2026

Checkpoint 2 is your **class skeleton plus a running program**: planned classes in code, and a basic game loop you can build on.

See the [final project handout](https://csci1300.github.io/project) for full requirements.

## Requirements

Push to the **same public GitHub repository** you used for checkpoint 1.

1. **At least 3 classes**, each with a `.h` and `.cpp` file.
2. Each **header file** must include the class declaration, **member variables**, and **function prototypes** for member functions you expect to use.
3. Each **source file** must include the matching header and define those functions (implementations may be partial or stubby, but should not be blank files).
4. A **`main` function** and project that **compile and run** with `g++` (same style as homework).
5. A **game loop** and **main menu** the player can interact with (menu options may print placeholders or do minimal work).
6. Update **`README.md`** with a **Theme** section (2–4 sentences describing your game theme).

You do not need a finished map, bundle, four characters, Joja system, inventory, resource limits, or win/loss logic for checkpoint 2.
