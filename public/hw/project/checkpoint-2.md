# Final Project: Checkpoint 2

**Due:** Wednesday, July 8, 2026

Checkpoint 2 is your **class skeleton plus a running program**: planned classes in code, and a basic game loop you can build on.

See the [final project handout](https://csci1300.github.io/project) for full requirements.

## Requirements

Push to the **same public GitHub repository** you used for checkpoint 1.

1. **At least 3 classes**, each with a `.h` and `.cpp` file.
2. At least **one class must inherit from another class** using C++ inheritance.

   * For example, you might have a general `Character` class and a more specific `Farmer`, `Villager`, `Monster`, or `Shopkeeper` class that inherits from it.
   * The inheritance does not need to be fully complex yet, but it should be meaningful for your project design.
3. Each **header file** must include the class declaration, **member variables**, and **function prototypes** for member functions you expect to use.
4. Each **source file** must include the matching header and define those functions (can be incomplete).
5. A **`main` function** and project that **compile and run** with `g++` using the same style as homework.
6. A **game loop** and **main menu** the player can interact with.
   * Menu options may print placeholders or do minimal work for now.
7. Update **`README.md`** with a **Theme** section.
   * This should be 2–4 sentences describing your game theme.

You do not need a finished map, bundle, four characters, Joja system, inventory, resource limits, or win/loss logic for checkpoint 2.

However, your code should show that you have started planning your project structure, including how at least one class can specialize or extend another class through inheritance.
